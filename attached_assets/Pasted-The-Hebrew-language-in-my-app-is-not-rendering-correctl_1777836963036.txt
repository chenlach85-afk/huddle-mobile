The Hebrew language in my app is not rendering correctly. Fix it. Don't change features, only fix the Hebrew display and language handling.
Required changes
1. Languages
Support exactly three languages: Hebrew (he), English (en), Spanish (es). Hebrew is RTL; English and Spanish are LTR.
2. Fonts
Load Heebo from Google Fonts with weights 400, 500, 700, and 900. Add to the root layout's <head>:
html<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Heebo:wght@400;500;700;900&display=swap" rel="stylesheet">
Set Heebo as the primary font for ALL Hebrew text including display/headlines. Do NOT use Bebas Neue, Impact, Oswald, or any other Latin-only display font when the active language is Hebrew — those fonts have no Hebrew glyphs and will render as broken boxes or fall back to ugly defaults.
In your CSS / Tailwind config:
css:root {
  --font-body: 'Heebo', 'Inter', system-ui, sans-serif;
  --font-display-en: 'Bebas Neue', 'Oswald', 'Impact', sans-serif;
  --font-display-he: 'Heebo', sans-serif;
}

[lang="he"] body, [lang="he"] * {
  font-family: var(--font-body);
}

[lang="he"] .display-text {
  font-family: var(--font-display-he);
  font-weight: 900;
}
The only place Bebas Neue stays in Hebrew is the literal "HUDDLE" wordmark logo (because it's English Latin letters by design).
3. RTL handling
When the active language is Hebrew, set <html dir="rtl" lang="he">. When it's English or Spanish, set <html dir="ltr" lang="en"> or <html dir="ltr" lang="es">.
Use Tailwind's logical properties throughout (don't use ml-*, pr-*, text-left, text-right):

ms-* instead of ml-*
me-* instead of mr-*
ps-* instead of pl-*
pe-* instead of pr-*
start-* / end-* for absolute positioning
text-start / text-end

For direction-implying icons (back arrows, chevrons, "next" arrows), add a flip-rtl utility:
css[dir="rtl"] .flip-rtl { transform: scaleX(-1); }
Apply className="flip-rtl" to those specific icons.
4. Hebrew typography rules
These are non-negotiable:

No letter-spacing on Hebrew text. Strip any tracking-* or letter-spacing from elements that render Hebrew. Wide tracking visually breaks Hebrew letters apart and looks broken.
No uppercase styling on Hebrew. Remove uppercase / text-transform: uppercase from elements that render Hebrew (Hebrew has no uppercase form, and the CSS just makes it render slightly weird).
Hebrew text reads visually larger than equivalent English at the same px size. Bump body text from 13-14px (English) up to 14-16px when in Hebrew.
Numbers stay LTR even inside Hebrew text. Wrap any number that appears inside a Hebrew sentence with dir="ltr":

jsx  <span dir="ltr" style={{ unicodeBidi: 'embed' }}>17:00</span>
This applies to: jersey numbers, times, dates, scores, prices, phone numbers, percentages.
If your section labels currently use this style for English:
jsx<div className="text-[10px] tracking-[2px] font-bold uppercase">
For Hebrew they should be:
jsx<div className="text-[11px] font-bold">
(no tracking, no uppercase, slightly larger size)
5. Date and time
Install date-fns if not already present, plus the he and es locales.
jsimport { format } from 'date-fns';
import { he, enUS, es } from 'date-fns/locale';

const localeMap = { he, en: enUS, es };
const locale = localeMap[currentLang] || enUS;
Use these formats:

Hebrew: dates as DD/MM/YYYY, times always 24h (e.g., "17:00", not "5:00 PM")
English: M/D/YYYY, 12h or 24h based on user pref
Spanish: DD/MM/YYYY (Spain/LatAm convention), 24h

Hebrew day-of-week single letters (for compact day display): format(date, 'EEEEEE', { locale: he }) returns א׳ ב׳ ג׳ ד׳ ה׳ ו׳ ש׳.

When changed, immediately re-render with the new locale + direction without requiring a page reload.
8. PWA manifest
Make sure manifest.json declares Hebrew as the default:
json{
  "lang": "he",
  "dir": "rtl"
}
9. Default language detection
When a brand-new user (no saved preference yet) opens the app:

Check profiles.preferred_language if logged in
Otherwise check browser navigator.language:

he, he-IL → Hebrew
es, es-* → Spanish
anything else → English


Default fallback: Hebrew (if you want Hebrew-first), or English (if you want broader appeal). Pick one and be consistent.
