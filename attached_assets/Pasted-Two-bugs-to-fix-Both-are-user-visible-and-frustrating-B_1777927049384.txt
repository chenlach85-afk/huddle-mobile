typescript// Wrong:

Fix: ensure setLoading(false) runs after the await — preferably in a finally block:
typescriptconst handleSave = async () => {
  setLoading(true);
  try {
    const { error } = await supabase.from('profiles').update(data).eq('id', userId);
    if (error) throw error;
    showSuccess('Saved');
  } catch (err) {
    showError('Save failed: ' + err.message);
  } finally {
    setLoading(false);
  }
};
Cause B: Supabase call is silently failing
If RLS or a constraint blocks the update, the call returns an error but the code only checks for success. Always check error:
typescriptconst { data, error } = await supabase.from('profiles').update(...).eq('id', userId);
if (error) {
  console.error('Save error:', error);
  setLoading(false);
  showError(error.message);
  return;
}
Check the browser network tab and console for the actual response. There might be an RLS policy issue (e.g., the user can't update certain fields) or a constraint violation (e.g., the value doesn't match the check constraint).
Cause C: Optimistic update racing with server response
If the UI updates locally first then refetches, a slow refetch makes it look frozen. Decouple the local state update from the refetch.
Cause D: Toast/banner doesn't auto-dismiss
If you're using a toast that says "Saving..." and you only update its message but never dismiss it, it stays forever. Use a toast library that auto-dismisses success messages after 3-5 seconds, or call dismiss() explicitly:
typescriptconst toast = showToast('Saving...', { persistent: true });
try {
  await save();
  toast.update('Saved!', { duration: 3000 });
} catch (e) {
  toast.update('Failed', { type: 'error' });
}
Cause E: Form library state not resetting
If using react-hook-form, after a successful submit, the isSubmitting flag should clear automatically. If it doesn't, you might be using setValue outside the form context or have a stale handler.

Add a global save-state pattern (optional but worth it)
To prevent this bug from coming back in other settings forms, create a reusable hook:
typescript// hooks/useSaveState.ts
export function useSaveState() {
  const [state, setState] = useState<'idle' | 'saving' | 'saved' | 'error'>('idle');
  const [error, setError] = useState<string | null>(null);

  const save = async (fn: () => Promise<void>) => {
    setState('saving');
    setError(null);
    try {
      await fn();
      setState('saved');
      setTimeout(() => setState('idle'), 3000);
    } catch (err: any) {
      setState('error');
      setError(err.message || 'Unknown error');
      setTimeout(() => setState('idle'), 5000);
    }
  };

  return { state, error, save };
}
