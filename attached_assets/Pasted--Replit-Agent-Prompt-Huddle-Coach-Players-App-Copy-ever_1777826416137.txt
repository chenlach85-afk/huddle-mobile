# Replit Agent Prompt — Huddle (Coach + Players App)

Copy everything below the line into Replit Agent's "Describe your idea" box. Turn ON Plan mode before submitting.

---

Build a mobile-first PWA called **Huddle** for sports team management. Tagline: *"The whole team. One huddle."* (Hebrew: "כל הקבוצה. במקום אחד.")

One app, two roles: **Coach** (manages everything) and **Player** (views schedule, confirms attendance, completes tasks). Generic for any sport. Bilingual: English + Hebrew with full RTL support and a language switcher.

## Design philosophy — sport-team aesthetic

This is NOT a generic SaaS app. It must feel like a sports product. Specific rules:

- **Stadium night vibe**: deep navy base (not slate gray), warm energetic accents
- **Per-team theming**: each coach picks their team color during onboarding; the app's accent re-themes to that color across all screens
- **Jersey number identity**: players are represented by their jersey number in a colored tile, not a generic avatar circle
- **Status as scoreboard stats**: "12 IN · 3 OUT · 5 ?" not "12 confirmed, 3 declined, 5 pending"
- **Bold condensed typography for display**: Impact / Bebas Neue / Oswald for headers, numbers, and titles. Jersey-number style.
- **Energy in hero cards**: gradient backgrounds with abstract circular shapes (like stadium scoreboards), not flat plain cards
- **Action-oriented language**: "Squad" not "Roster" anywhere users see; "Lineup" for upcoming events; "Huddle" for the home screen

## Tech stack

- **Framework**: Next.js 14 with App Router, TypeScript, Tailwind CSS
- **PWA**: next-pwa with offline support, installable to home screen, push notifications
- **Database & Auth**: Supabase (Postgres + Auth + Storage)
- **Realtime**: Supabase Realtime for messages and live updates
- **Internationalization**: next-intl for Hebrew/English with RTL handling
- **Push notifications**: Web Push API (free) — fallback to email via Resend
- **Icons**: lucide-react
- **Forms**: react-hook-form + zod
- **Date handling**: date-fns with Hebrew locale support
- **Fonts**: Inter (body), Bebas Neue or Oswald (display, English), Heebo Black (display, Hebrew)

## Design system — exact tokens

### Color palette


**Per-team color theming**: At app boot, after loading the user's active team, set `document.documentElement.style.setProperty('--team-color', team.primary_color)`. All accent surfaces (the home hero card gradient, the active tab indicator, jersey number tiles for the player viewing) use `var(--team-color)` instead of the default ignition orange. Default fallback is `#FF6B35` if a team has no color set.

### Typography

```css
/* Font families */
--font-display: 'Bebas Neue', 'Oswald', 'Impact', 'Heebo', sans-serif; /* condensed display */
--font-body: 'Inter', 'Heebo', system-ui, -apple-system, sans-serif;
--font-numeric: 'Bebas Neue', 'Oswald', monospace; /* jersey numbers, scores */
```

For Hebrew: use **Heebo** for body and **Heebo with weight 900** for display (Heebo has a beautiful condensed-bold feel that pairs visually with Bebas Neue).

**Sizing rules**:
- Display headlines (TUE · 17:00, scoreboard stats): 22-28px, font-display, font-weight 900, letter-spacing -1px
- Section labels (small caps): 10-11px, letter-spacing 1.5-2px, font-weight 700, uppercase, color text-faint
- Body: 13-15px, font-body, font-weight 400-500
- Jersey numbers: font-numeric, weight 900, white text on team-color tile

### Component aesthetic

**Cards**:
- Standard: `bg-elev` background, `border-radius: 12px`, `padding: 14-16px`, no shadow
- Hero: gradient from team color to a darker shade, `border-radius: 16px`, decorative circular shapes positioned absolute (top-right and bottom-left) with low opacity
- Activity row: `bg-elev` with `border-left: 3px solid` in a status color (green/red/blue)

**Buttons**:
- Primary action: solid `var(--team-color)` background, white text, `border-radius: 12px`, `padding: 14px 20px`, font-weight 600, uppercase if short
- Secondary: outline with `border-color: rgba(255,255,255,0.2)`, transparent bg, hover bg `bg-elev`
- Quick action squares: 22x22px colored icon tile in a card with 8-9px label below

**Player tiles** (jersey number representation):
- 40x40px rounded square (`border-radius: 12px`)
- Linear gradient background using the team's color (e.g., `linear-gradient(135deg, #FF6B35, #C73E1D)`)
- Jersey number centered, font-numeric, weight 900, color white
- Status dot in top-right corner: 10x10px circle, color by status (green=confirmed, red=declined, yellow=pending), 2px navy border (matches bg)
- Player name below tile, 8-10px, weight 600

**Bottom tab bar** (mobile primary nav):
- Fixed bottom, height 64px, `bg: rgba(0,0,0,0.4)` with `backdrop-blur(12px)`, `border-top: 0.5px solid rgba(255,255,255,0.08)`
- 5 equal tabs
- Active tab: 18x18px solid `var(--team-color)` square icon, label below in `var(--team-color)`, weight 700, letter-spacing 0.5px
- Inactive tab: 18x18px outline icon (1.5px border, white 40% opacity), label `text-faint` weight 600

**Modals**:
- Slide-up sheet on mobile (full-width, rounded top corners 24px, `bg-elev` background)
- Centered modal on desktop (max-width 560px, rounded 16px, backdrop blur)

**Forms**:
- Inputs: `bg-elev-2` background, no border, `border-radius: 10px`, `padding: 14px 16px`, focus ring 2px team color
- Labels above inputs, 11px, letter-spacing 1px, weight 700, uppercase, `text-faint`

### Mobile-specific

- Touch targets ≥ 44px
- Bottom tab bar always visible
- Modals slide up from bottom
- Generous padding (16-20px) inside cards
- Avoid hover-only interactions
- High contrast — coaches use this in bright outdoor sun

### RTL handling

- Use Tailwind logical properties: `ms-` (margin-inline-start) instead of `ml-`, `pe-` instead of `pr-`
- Text alignment: `text-start` / `text-end`, never `text-left` / `text-right`
- Icons that imply direction (back arrow, forward chevron) flip in RTL
- Numbers stay LTR even in Hebrew context (jersey numbers, times, scores)
- Set `<html dir="rtl" lang="he">` dynamically based on user's preferred language

```sql
create index events_team_starts_idx on events(team_id, starts_at);

create table public.attendance (

  status text default 'pending' check (status in ('pending', 'confirmed', 'declined', 'maybe', 'attended', 'no_show')),

  file_url text,

  created_at timestamptz default now()

-- ============================================================================
-- AUTO-CREATE PROFILE on signup
-- ============================================================================
create or replace function public.handle_new_user()
returns trigger language plpgsql security definer set search_path = public
as $$
begin
  insert into public.profiles (id, email, full_name)
  values (new.id, new.email, coalesce(new.raw_user_meta_data->>'full_name', ''));
  return new;
end; $$;

drop trigger if exists on_auth_user_created on auth.users;
create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure public.handle_new_user();

-- ============================================================================

create or replace function public.is_team_member(team_uuid uuid)
returns boolean language sql security definer stable as $$
  select exists(
    select 1 from team_members
    where team_id = team_uuid and user_id = auth.uid() and status = 'active'
  );
$$;

create or replace function public.is_team_coach(team_uuid uuid)
returns boolean language sql security definer stable as $$
  select exists(
    select 1 from team_members
    where team_id = team_uuid and user_id = auth.uid()
      and role in ('coach', 'assistant_coach') and status = 'active'
  );
$$;

create policy "view own and teammates profiles" on profiles for select
  using (
    id = auth.uid() or
    exists(select 1 from team_members tm1
           join team_members tm2 on tm1.team_id = tm2.team_id
           where tm1.user_id = auth.uid() and tm2.user_id = profiles.id)
  );
create policy "update own profile" on profiles for update using (id = auth.uid());

create policy "members view team" on teams for select using (is_team_member(id));
create policy "coaches update team" on teams for update using (is_team_coach(id));
create policy "anyone create team" on teams for insert with check (created_by = auth.uid());

create policy "members view roster" on team_members for select using (is_team_member(team_id));
create policy "coaches manage roster" on team_members for all using (is_team_coach(team_id));
create policy "users join team" on team_members for insert with check (user_id = auth.uid());

create policy "members view events" on events for select using (is_team_member(team_id));
create policy "coaches manage events" on events for all using (is_team_coach(team_id));

create policy "view own attendance" on attendance for select
  using (user_id = auth.uid() or
         exists(select 1 from events e where e.id = attendance.event_id and is_team_coach(e.team_id)));
create policy "users update own attendance" on attendance for all using (user_id = auth.uid());
create policy "coaches manage attendance" on attendance for all
  using (exists(select 1 from events e where e.id = attendance.event_id and is_team_coach(e.team_id)));

create policy "members view conversations" on conversations for select
  using (exists(select 1 from conversation_members cm where cm.conversation_id = id and cm.user_id = auth.uid()));
create policy "coaches create conversations" on conversations for insert with check (is_team_coach(team_id));
create policy "members view conv members" on conversation_members for select
  using (exists(select 1 from conversation_members cm where cm.conversation_id = conversation_members.conversation_id and cm.user_id = auth.uid()));
create policy "coaches manage conv members" on conversation_members for all
  using (exists(select 1 from conversations c where c.id = conversation_id and is_team_coach(c.team_id)));

create policy "members send messages" on messages for insert
  with check (sender_id = auth.uid() and
              exists(select 1 from conversation_members cm where cm.conversation_id = messages.conversation_id and cm.user_id = auth.uid()));

create policy "members view announcements" on announcements for select using (is_team_member(team_id));
create policy "coaches manage announcements" on announcements for all using (is_team_coach(team_id));
create policy "users mark read" on announcement_reads for all using (user_id = auth.uid());

create policy "members view programs" on training_programs for select using (is_team_member(team_id));
create policy "coaches manage programs" on training_programs for all using (is_team_coach(team_id));

create policy "members view team tasks" on tasks for select using (is_team_member(team_id));
create policy "coaches manage tasks" on tasks for all using (is_team_coach(team_id));

create policy "view own assignments" on task_assignments for select
  using (user_id = auth.uid() or
         exists(select 1 from tasks t where t.id = task_id and is_team_coach(t.team_id)));
create policy "users update own assignment status" on task_assignments for update using (user_id = auth.uid());
create policy "coaches manage assignments" on task_assignments for all
  using (exists(select 1 from tasks t where t.id = task_id and is_team_coach(t.team_id)));

create policy "members view files" on team_files for select using (is_team_member(team_id));
create policy "members upload files" on team_files for insert
  with check (uploaded_by = auth.uid() and is_team_member(team_id));
create policy "uploader or coach delete" on team_files for delete
  using (uploaded_by = auth.uid() or is_team_coach(team_id));
```

## App structure

### Public routes
- `/` — Landing: bold hero with "HUDDLE" wordmark, tagline, "Sign in" / "Sign up" / "Join with code" CTAs
- `/auth/signin` — Email + password + Google OAuth
- `/auth/signup` — Email + password + name
- `/auth/forgot-password`
- `/join/[code]` — Player joins via invite code

### Authenticated routes
- `/onboarding` — First-run: choose role (coach creates team OR player joins via code)
- `/teams` — List user's teams
- `/team/[id]` — Team home (different view per role)
- `/team/[id]/schedule` — Calendar of events
- `/team/[id]/event/[eventId]` — Event detail
- `/team/[id]/messages` — Group chat + DMs
- `/team/[id]/announcements` — Announcements feed
- `/team/[id]/squad` — Player roster (was "roster" — use "squad" everywhere)
- `/team/[id]/training` — Training programs library
- `/team/[id]/tasks` — Tasks
- `/team/[id]/files` — Photo & file gallery
- `/team/[id]/settings` — Coach: edit team info, manage invites, change team color
- `/profile` — User profile, language, notifications

```json
    {"src": "/icon-512.png", "sizes": "512x512", "type": "image/png"}
  ]
```

Service worker caches static assets and last-loaded data for offline read. Show "OFFLINE — last synced [time]" banner when disconnected.

Push notifications trigger on: new event, new message, new announcement, new task, event reminder (24h before).

## API routes

- `POST /api/teams` — create team (becomes coach)
- `POST /api/teams/[id]/join` — join via invite code
- `POST /api/events` — create event + push to team
- `POST /api/events/[id]/attendance` — RSVP
- `POST /api/messages` — send message + push
- `POST /api/announcements` — create + push
- `POST /api/tasks` — create + assign + push
- `POST /api/tasks/[id]/complete` — mark complete (with proof)
- `POST /api/upload` — upload to Supabase Storage
- `POST /api/push/subscribe` — save user's push subscription
- `POST /api/push/send` — internal helper

## Critical rules

1. **Sport-team aesthetic, not corporate**: dark navy bg, bold condensed display fonts, jersey number tiles, scoreboard-style hero card, energetic accent colors
2. **Per-team color theming**: every accent surface uses `var(--team-color)` set dynamically from `teams.primary_color`
3. **Bilingual with full RTL**: Hebrew flips correctly throughout, including icons that imply direction
4. **Role-based UI**: same routes, different layouts for coach vs player
5. **Realtime messages**: Supabase Realtime subscription on messages table
6. **Push notifications**: working web push end-to-end
7. **File uploads**: photos, videos, PDFs into Supabase Storage with proper RLS
8. **Offline read**: cached events/tasks/messages visible offline
9. **Invite via code**: 8-character codes, deep-linkable, shareable via WhatsApp

## Acceptance criteria

App works when:
1. Coach signs up, creates team with custom color, gets invite code
2. App is themed in coach's chosen team color (visible across all accents)
3. Coach adds an event with all fields (uniform, arrival time, what to bring) — players get push
4. Player signs up via invite link, joins team
5. Player sees event with RSVP buttons, taps "I'M IN" — coach sees status update in real-time
6. Coach sees scoreboard stats "X IN · Y OUT · Z ?" updating live
7. Coach sends announcement — players get push, see it on home
8. Coach uploads training PDF, creates task linked to it, assigns to all players
9. Player marks task complete with photo proof — coach sees completion
10. Group chat works real-time
11. Player switches language to Hebrew — full UI mirrors RTL correctly, fonts swap to Heebo
12. App installs to home screen, opens fullscreen, works offline (read-only) for cached data
13. Two teams created with different colors — each coach sees their app in their team's color, no leakage
14. RLS prevents cross-team data access (test: create 2 teams, verify isolation)

## What NOT to build

- Payment / paid plans (later phase — billing addendum)
- Video calling
- Live game scoring
- Statistics dashboards beyond simple attendance %
- Social features (likes, comments)
