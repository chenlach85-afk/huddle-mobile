Light mode has invisible text in places. White text is hardcoded in many components and stays white even in light mode, making it disappear on the light background. Find every hardcoded white-ish color and replace with theme-aware tokens.
The exact problem
The CSS variables work correctly for backgrounds and most text. But some components use:

className="text-white"
className="text-[#fff]" or text-[#ffffff]
className="text-[rgba(255,255,255,...)]"
Inline styles like style={{ color: 'white' }} or style={{ color: '#fff' }}
Any explicit white color value

These don't change with the theme. They render white on white in light mode = invisible.

Step 2 — Decide the right replacement for each
The replacement depends on context. Here's the decision tree:
If the white text is on a permanently dark background
Examples: text inside the orange gradient hero card, text inside a colored status pill (green/red/blue), text inside a dark-themed modal that stays dark in both modes, the HUDDLE wordmark.
These should stay white because they sit on a colored or dark surface that exists in both themes. But mark them explicitly so it's clear they're intentional:

Either keep text-white (it's correct for these cases)
Or use a more explicit class like text-on-accent (a new utility for "text that appears on top of a colored brand surface")

The orange gradient hero card text is the main example: orange is orange in both themes, so white text on orange always works.
If the white text is supposed to mean "primary readable text"
Examples: card titles, body text, list item text, form labels, headers, navigation labels, anything that's "the main text content."
Replace with a theme-aware token:

text-white → text-text (which maps to var(--text) — white in dark mode, deep navy in light mode)

If the text is "secondary/dimmed text"
Examples: timestamps, hints, captions, helper text under inputs, "Sent 2 hours ago" type labels.
Replace with:

text-[rgba(255,255,255,0.7)] → text-text-dim (var(--text-dim))
text-[rgba(255,255,255,0.5)] → text-text-faint (var(--text-faint))
text-[rgba(255,255,255,0.3)] → text-text-ghost (var(--text-ghost))

Adding the text-on-accent utility (optional but clean)
If you want explicit semantics for "text on colored surface," add this to Tailwind config:
javascript{
  theme: {
    extend: {
      colors: {
        // ...existing tokens
        'on-accent': '#ffffff',  // always white, intentionally
      }
    }
  }
}
Then use text-on-accent for the orange card text, status pill text, etc. This makes it easy in the future to find "intentional white" vs "accidental white."
